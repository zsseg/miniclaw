# **Clawmini 可视化应用使用文档（用户版·详细）**

## **一、文档用途**

这份文档面向**最终用户**，用于帮助你在不看代码的情况下，完成以下工作：

1. 与智能体进行对话交互；
2. 使用 3.2 自动撰写文稿（含 2~3 方案推荐）；
3. 使用 3.1 QQ 自动回复调试（配置、状态、轮询、取消）；
4. 使用 3.3 批量图片处理（执行、状态查询、取消）；
5. 查看输出文件与常见问题处理方式。

---

## **二、功能总览（你能做什么）**

### **2.1 一句话概览**

`Clawmini` 是一个“可视化智能体工具台”，你可以在一个界面里完成文本任务、QQ自动回复调试、图片批处理和结果预览。

### **2.2 六大功能模块**

1. **聊天问答**
  - 输入自然语言任务，系统自动规划并执行。
2. **规划轨迹展示**
  - 可查看每轮 `Thought → Action → Observation`。
3. **3.2 文稿方案推荐**
  - 对开放式写作请求先给出 2~3 个候选方案。
4. **文稿生成与预览**
  - 选定方案后一键生成，并在预览页直接查看。
5. **3.1 QQ 自动回复调试**
  - 配置群号、私聊延迟，模拟消息，轮询待回复，取消任务。
6. **3.3 图片批量处理**
  - 调整尺寸、转换格式、查询任务进度、取消任务。

---

## **四、界面结构详解**

### **4.1 左侧：会话区**

用途：输入任务、查看回复。当前左侧会话区固定在页面左边，约占整页三分之一，右侧会自动扩展给 QQ / 文稿 / 图片页。

支持：
- 普通发送：点击 **发送消息**
- 快捷键发送：`Ctrl + Enter`
- 清空显示：点击 **清空聊天**（不删除历史文件）
- 如果左侧内容太多，可以直接滚动查看，不需要再把底部控制台当成输入区。

### **4.2 右侧页签：规划轨迹**

用途：观察系统每一步在做什么。

你会看到：
- 第几轮
- 选择了哪个工具
- 工具返回了什么结果

适用场景：
- 结果不符合预期时用于定位原因；
- 调整提示词前先看系统实际执行路径。

### **4.3 右侧页签：方案选择（3.2）**

用途：处理开放式写作需求。

展示内容：
- 方案主题
- 风格
- 目标字数
- 方案说明（为什么这样写）

### **4.4 右侧页签：文稿预览**

用途：查看已经生成的文稿。

展示内容：
- 当前文稿文件路径
- 文稿全文

按钮：
- **刷新预览**：重新从磁盘读取文件内容。

### **4.5 右侧页签：3.1 QQ自动回复**

用途：调试 QQ 自动回复策略。

可做操作：
- 先点 **下载NapCat** 打开发布页下载安装包。
- 安装后点 **一键启动/填充**，自动发现 NapCat、账号 ID 和 API 基址。
- 再点 **更新配置** 保存群号、私聊延时、冷却和账号 ID。
- 点 **查询状态** 查看 `gateway_mode`、`binding`、`pending_private`。
- 用 **模拟收到消息** 验证群聊 @ 和私聊等待队列。
- 用 **轮询待回复** 触发到期私聊发送。
- 用 **一键绑定客户端** 或 **恢复/暂停** 调整本地绑定与自动回复状态。
- 运行后右下状态区会显示 **NapCat事件接收地址**，把 NapCat 的事件回调指向这个地址后，app 才能真正收到消息并自动回复。
- 如果你只看见“发送成功”但 app 没有反应，先确认 NapCat 是否把消息事件 POST 到本地接收地址，而不是只调用了发送接口。

### **4.6 右侧页签：3.3 图片处理**

用途：批量处理图片并管理任务。

可做操作：
- 执行图片任务
- 查询任务状态（progress）
- 取消任务

## **五、核心功能详细使用说明**

###[1] 聊天问答与规划轨迹

**输入**：任意自然语言任务（如“帮我写文稿”）。

**输出**：

###[2] 3.2 自动撰写文稿（方案推荐→生成→预览）

### A. 方案推荐

1. 在输入框输入开放式请求，例如：
  - `帮我写一篇人工智能伦理短文`
2. 点击 **开放式方案(3.2)**。
3. 右侧生成 2~3 个方案卡片。

### B. 选择并生成

1. 点击 **选择方案X**。
2. 点击 **按当前选中方案生成文稿**。
3. 会话区显示生成结果；文稿预览页自动加载文件。

### C. 结果说明

- 生成文件默认在 `workspace/`
- 文件扩展名限制为 `.md/.txt`

---

###[3] 3.1 QQ 自动回复（详细）

### A. 配置参数说明

- **目标群号**：仅该群触发群聊自动回复逻辑。
- **私聊延时(秒)**：私聊消息在该时间后才允许自动回复。
- **自定义提示词**：用于调整回复风格（例如“简洁回答”）。
- **网关模式**：`mock` 仅演示，`managed` 走 NapCat 在线发送，`windows` 走本地窗口/在线发送混合流程。

### B. 标准操作流程

1. 填好参数，点击 **更新QQ配置**。
2. 先看状态区里的 **NapCat事件接收地址**，把它填到 NapCat 的事件回调配置里。
3. 点击 **查询QQ状态**，确认：
  - enabled / paused
  - gateway
  - pending_private（待回复队列数）
  - binding（NapCat / 本地绑定反馈）
4. 在“消息模拟”输入来源、chat_id、sender_id、文本，确认策略本身没问题。
5. 真正联调时，让 NapCat 把群聊/私聊事件 POST 到本地接收地址。
6. 对私聊任务，点击 **轮询待回复** 触发到期发送。
7. 若用户已手动回复，点击 **模拟用户手动回复(取消)**。

### C. 状态与结果解释

- `pending_private = 0`：当前没有待发送私聊。
- 如果你刚模拟了私聊但还是 0，先检查 **启用私聊回复** 是否勾选，再确认 `私聊延时(秒)` 和“消息模拟”里的 `waited_sec` 是否小于延时。
- “尚未到达等待阈值”：已入队等待，不是失败。
- “已取消待回复私聊任务”：手动回复覆盖生效。
- `binding` 里如果出现 `notes`，就是当前网关/ NapCat 的绑定反馈；`connected=True` 通常表示已成功连上。

###[4] 3.3 批量图片处理（详细）

### A. 参数说明

- **输入目录**：相对 `workspace` 的图片目录（如 `images`）
- **输出目录**：相对 `workspace` 的输出目录（如 `output`）
- **格式**：`png/jpg/webp`
- **宽/高**：目标尺寸
- **保持比例**：开启后按比例缩放

### B. 标准操作流程

1. 设置参数后点击 **执行图片任务**。
2. 系统返回 `task_id`（会自动写入任务框）。
3. 点击 **查询任务状态**，查看：
  - `status`：running/completed/canceled
  - `progress`：`processed/total`
4. 如需终止，点击 **取消任务**。

### C. 任务状态解释

- `running`：任务处理中
- `completed`：任务完成
- `canceled`：任务已取消

## **八、常见问题（FAQ）**

### Q1：提示“请先选择一个方案”

先执行 **开放式方案(3.2)**，再点击某个 **选择方案X**。

### Q2：文稿预览为空

先确认会话区是否显示生成成功，再点击 **刷新预览**。

### Q3：QQ 私聊没有马上回复

这是正常现象。私聊受延时阈值控制，到期后请点击 **轮询待回复**。

### Q4：图片任务找不到进度

请先执行任务并确认 `task_id` 已填入，然后点击 **查询任务状态**。

### Q5：能否直接导出 PDF

当前默认输出 `.md/.txt`，可使用本地工具转换为 PDF。

## **十一、QQ 内容读取/修改范围与安全性报告（用户可读）**

本节说明：应用在 3.1「QQ 自动回复」能力中，**可能读取什么**、**可能修改什么**、以及对应的安全控制。

### **11.1 可能读取的 QQ 内容（Read）**

仅用于自动回复决策，读取范围如下：

1. **消息元信息**
   - 来源类型：群聊 / 私聊
   - 会话标识：`chat_id`
   - 发送者标识：`sender_id`（内部会脱敏）
   - 消息时间（用于私聊延时判断）

2. **群聊触发相关信息**
   - 是否明确 @ 当前账号（`mentioned`）
   - 是否为 `@所有人`（`mention_all`）
   - 目标群号是否匹配配置

3. **消息正文（文本）**
   - 用于生成回复前的语义理解
   - 会先进行注入清洗（如可疑提示词过滤）

4. **私聊等待队列状态**
   - 是否已有待回复任务
   - 用户是否在等待窗口内手动回复（用于取消自动发送）

> 当前规则已明确：**群 `@所有人` 不触发自动回复**。

### **11.2 可能修改的 QQ 内容（Write）**

应用不会修改 QQ 客户端数据库；“修改”仅指自动发送行为与本地控制状态变化：

1. **向聊天窗口发送文本**
   - 场景：满足触发条件后自动回复
   - 限制：回复长度截断（≤200字）

2. **本地任务状态变化**
   - 新增/删除私聊待回复任务（`pending_private`）
   - 更新冷却计时（防重复发送）

3. **本地日志写入**
   - 文件：`workspace/qq_auto_reply.log`
   - 内容：时间、会话、消息摘要、回复摘要（用于审计）

### **11.3 安全性评估（按风险面）**

#### A. 指令注入风险（中）

- 风险示例：消息中夹带“忽略之前指令”等诱导文本。
- 当前防护：
  - 入站消息先经过注入清洗；
  - 回复模板固定且有长度约束；
  - 群消息必须满足明确触发条件。
- 建议增强：
  - 增加更完整的关键词与模式黑名单；
  - 在提示层强制“仅回答业务问题，不执行消息内指令”。

#### B. 隐私泄露风险（中）

- 风险示例：将真实 QQ 标识直接写入外部上下文。
- 当前防护：
  - 日志中使用脱敏 sender（如 `user_1234`）；
  - 只记录消息摘要，不记录全文。
- 建议增强：
  - 日志增加自动脱敏与定期清理策略；
  - 对高敏感关键词（手机号/证件号）做掩码处理。

#### C. 误触发与刷屏风险（低~中）

- 风险示例：群里频繁触发导致连续回复。
- 当前防护：
  - 非目标群忽略；
  - 群 `@所有人` 忽略；
  - 冷却时间限制（默认 30 秒）；
  - 支持安静模式 `pause`。
- 建议增强：
  - 增加分联系人速率限制；
  - 增加每日发送上限。

#### D. 误发风险（私聊窗口）

- 风险示例：用户已手动回复后仍自动发送。
- 当前防护：
  - 支持 `user_replied` 取消待发送任务；
  - `poll_pending` 仅处理到期且未取消任务。
- 建议增强：
  - 与真实 QQ 发送记录做双重校验。

### **11.4 结论（当前可用级别）**

- 在现有实现中，3.1 已具备“**可调试可审计**”的基础安全控制；
- 对你提出的要求已落实：**群 `@所有人` 不接收、不触发回复**；
- 若用于真实生产场景，建议按 11.3 的增强项补齐风控与隐私治理。

### **11.5 模型如何识别“@我”而不是“@别人”**

核心原则：**先由 QQ 网关判定，再交给智能体处理**，不是让大模型靠文本“猜测”。

判定流程：

1. QQ 网关上报原始事件（包含 `mentions` 列表、`mention_all` 标记）；
2. 系统读取当前账号标识 `self_user_id`；
3. 按规则计算：
  - 若 `mention_all=true`：一律视为非 `@我`（并忽略自动回复）；
  - 否则仅当 `self_user_id` 在 `mentions` 中时，判定为 `@我`；
4. 将结果写入标准字段：
  - `mentioned`（是否明确 @ 我）
  - `mention_all`（是否 @所有人）
5. `qq_auto_reply` 再依据这两个布尔值决定是否触发回复。

可用公式表示：

$$
is\_mention\_me = (self\_user\_id \in mentions) \land (\lnot mention\_all)
$$

这能确保：

- `@其他用户` 不会误触发；
- `@所有人` 不会触发；
- 只有“明确 @ 当前账号”时才触发群自动回复。
